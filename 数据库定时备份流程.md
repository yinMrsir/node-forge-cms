# 数据库定时备份流程

## 1. 手动测试是否能正常备份

修改`server/bin/mysql_backup.sh`文件中的数据库密码，并执行备份脚本

```bash
sh server/bin/mysql_backup.sh
```

无报错、目录下出现 .sql.gz 压缩包即为成功。

## 2. 配置 Crontab 定时任务

```bash
crontab -e
```

### 示例常用规则

```bash
# 每天凌晨 2点 自动备份
0 2 * * * /your_path/server/bin/mysql_backup.sh >> /data/mysql_backup/backup_log.log 2>&1
```

crontab 格式：

`分 时 日 月 周 命令`

`0 2 * * *` 每天02:00

`0 1 * * 0` 每周日凌晨1点

`*/30 * * * *` 每30分钟

### 查看定时任务

```bash
crontab -l
```

## 3. 数据库恢复命令

```bash
# 解压
gunzip /server/bin/backup/mysql/your_database_*.sql.gz

# 解压并恢复
gunzip -c /server/bin/backup/mysql/your_database_*.sql.gz | mysql -u $DB_USER -p$DB_PASS $DB_NAME
```
